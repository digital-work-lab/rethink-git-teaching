3. Transfer between areas

  • First we created a setting for Transfer Knowledge Development:
    • Students explore questions involving concepts from different Git areas.
    • Utilizes diverse and unconnected problem sets that are challenging to combine into a coherent case exercise.
    • Proposes the use of hypothetical scenarios and an array of transfer questions.
  • Order of Git Areas:
    • Proceed from transfer questions related to branching and committing, to branching and collaborating, and finally to committing and collaborating.
    • Table 2 provides example questions for each transfer block and the rationale for their order.
  • Process of Solving Transfer Questions:
    • Provide students with a scenario and the transfer question.
    • Solutions may involve:
      • Intuition based on pre-existing knowledge of Git concepts.
      • Consultation of the official Git manual pages.
      • A web search.
      • Testing outcomes in an example repository.
    • Given the challenging nature, provide students with a detailed solution to revisit after the session.
  • Benefits of Connective Thinking Mode:
    • Helps students comprehend various Git issues that arise from multiple areas.
    • Necessary for deeper understanding of Git and rationalizing issues at the intersection of different areas.
    • Example: Rewriting branch histories in collaborative settings.
  • Holistic Understanding of Git:
    • Encourages thinking in a more connected way.
    • Helps tackle complex issues with a comprehensive perspective.
    • Enhances proficiency in using Git beyond just software development.
